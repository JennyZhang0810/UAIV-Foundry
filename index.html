Data Operating System for low-altitude dataset production

让低空数据集的构建、质检、发布和评测变成一条可审计的生产链。

UAIV-Foundry 不是另一个标注平台，也不只是资源清单。它是围绕低空多模态数据集生产而构建的数据操作系统，负责把 manifest、QA、release、dataset card、benchmark protocol 和 failure-to-data iteration 显式组织起来。

GitHub 返回 UAIV DataHub 查看关键入口
UAIV-Foundry 负责把数据产品、人工标注和发布评测组织成一套可追溯流程。

Foundry 在 UAIV 生态里扮演的不是“又一个页面”，而是后台操作系统。

它连接 UAIV-Real 和 UAIV-Labeler：前者是对外发布的数据产品，后者是人工生产工作台，而 Foundry 负责把数据整理、QA、release readiness、数据卡、benchmark request 和后续迭代做成系统化基础设施。

Knowledge Layer

知识与资源层

整理 annotation platforms、低空数据集、data quality、synthetic data 和 remote-sensing VLM 相关资源，为方法和流程选择提供可查证的背景库。

Data Factory Layer

数据工厂层

负责编排 manifest、worker batch、Golden Dataset、Labeler import、annotation export intake、QA 与 rework，把人和数据组织起来。

Release & Benchmark Layer

发布与评测层

负责 dataset card、release readiness、public package check、benchmark request protocol、metric specs 和 failure-to-data 闭环。

当前最重要的成果，不是概念，而是已经落地的 release candidate 组织能力。

围绕 UAIV-Real CityGov，Foundry 已经不再停留在“标注前准备”口径，而是能够对应当前的标准化任务包、release docs、split、audit 和 benchmark 协议。

以下能力都能在当前仓库和现有 release candidate 里找到对应证据，而不是停留在路线图口号。

Dataset packaging

  • task-wise `labels.json` / `labels.jsonl`
  • portable image package
  • metadata catalog and coverage report
  • dataset card / schema / taxonomy / benchmark protocol

QA and audit

  • exact duplicate audit
  • near-duplicate candidate audit
  • label vocabulary cleanup
  • bbox clipping policy and issue list

Split and benchmark

  • development split
  • grouped benchmark split
  • adapter protocol and metric specs
  • benchmark dry-run and future real benchmark handoff

Production orchestration

  • manifest generation
  • worker import packages
  • Golden QA templates
  • release checklist and project dashboard

后续方向

Foundry 的下一步不是再堆更多概念，而是继续把现有 release candidate 的最后几个公共缺口补齐。

Public release completion

补齐 DOI、public links、final license 和 privacy final counts，让当前数据产品真正完成公共沉积闭环。

Benchmark operationalization

把 current grouped split、adapter protocol 和 metric specs 进一步固化成可直接执行的 benchmark 流程。

Reusable Data OS

把当前围绕 CityGov 完成的流程推广成可迁移到更多 UAIV 数据产品的标准化操作系统。
