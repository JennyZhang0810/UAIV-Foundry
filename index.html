Definition

Foundry 到底是什么

UAIV-Foundry 不是另一个标注平台，也不是普通资源列表。它是围绕低空数据集构建过程的生产线、质检系统、发布系统和评测协议。

Knowledge Layer

结构化整理标注平台、UAV/遥感数据集、数据质量工具、遥感 VLM、合成数据资源，并记录 license/access 和 UAIV 相关性。

awesome resources quality grade

Data Factory Layer

生成 manifest、Golden Dataset、标注员分工、Labeler 导入包，并接收标注导出进入 QA 与返工闭环。

manifest QA gates

Release & Benchmark Layer

生成数据卡、Dashboard、发布检查、benchmark request 和 metric specs，使数据生产结果可复现、可评估、可迭代。

dataset card benchmark dry-run

Ecosystem

UAIV-Real、UAIV-Labeler、UAIV-Foundry 的关系

UAIV-Real

最终要交付的数据集产品。当前首发候选是城市治理低空图像数据，等待人工标注、复核和最终 QA 后公开。

Dataset / Product

UAIV-Labeler

人工标注工作台。工作人员在这里做场景、目标框、OCR、事件 QA、环境状态和复核导出。

Annotation Workbench

UAIV-Foundry

生产和质检系统。它决定数据如何被组织、标注如何被回收、问题如何被发现、发布是否可以继续。

Data Production OS

Closed Loop

从原始数据到可发布数据集的闭环

Foundry 的核心价值是把“收集图片”变成“可复现的数据生产过程”。Benchmark 不是终点，失败样本会反过来驱动下一轮数据规划。

Why It Matters

为什么 Foundry 有价值

不是只贡献数据，而是贡献生产数据的方法

Foundry 记录数据如何进入、如何标注、如何质检、如何返工、如何发布和如何评测，使数据集构建过程可以被复现和审计。

不是普通 Awesome List，而是可执行知识层

资源库中的工具、数据集和方法被映射到 UAIV 的生产任务、QA 门禁、benchmark 协议和后续数据迭代。

让质量问题可定位、可返工、可闭环

QA 输出不是笼统评价，而是具体 issue、affected images、rework items 和 release blockers。

让评测结果反哺下一轮数据设计

Benchmark failure analysis 将难样本、类别缺口和模型分歧转化为下一轮采样和标注优先级。
